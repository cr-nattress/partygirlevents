## Story Format

Each user story follows this structure:
- Title, priority, estimate, status
- Acceptance criteria (checkbox list)
- Technical notes
- Dependencies
- Definition of Done

## Priority Levels

- **P0 — Must Have:** Blocks launch. No negotiation.
- **P1 — Should Have:** Important for launch quality but site can ship without.
- **P2 — Nice to Have:** Enhances experience, can be deferred.
- **P3 — Future:** Planned for later phases.

## Story Points

- **1 pt:** Hours of work. Config change, copy update, single component.
- **2 pts:** Half a day. Small feature, simple page.
- **3 pts:** 1-2 days. Page with multiple sections, form with logic.
- **5 pts:** 3-5 days. Complex feature, multi-step flow, API integration.
- **8 pts:** 1-2 weeks. Major feature (quiz, AI chat, estimator).
- **13 pts:** 2-3 weeks. Large system (CMS setup, full AI pipeline).
